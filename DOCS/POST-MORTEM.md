# Lumain AI Doctor: Post-Mortem & Knowledge Base

## Инцидент №1: Конфликт роутов (Route Specificity)
**Проблема**: Ошибка `Error: You cannot define a route with the same specificity as an optional catch-all route`.
**Причина**: Одновременное существование `src/app/p/[id]/page.tsx` и `src/app/p/[id]/[[...slug]]/page.tsx`.
**Урок**: При переходе на catch-all роуты (`[[...]]` или `[...]`), родительский `page.tsx` должен быть удален.
**Лечение**: Использовать `Node.js fs.unlinkSync` для гарантированного удаления в Windows-окружении, если обычный `rm` дает сбой.

## Инцидент №2: SSR Serialization / Client Handlers
**Проблема**: Передача `onClick` или других функций в props из Server Component в Client Component.
**Причина**: Нарушение границ SSR.
**Урок**: Все интерактивные действия (кнопки скачивания, печати) должны быть инкапсулированы внутри Client Components (`"use client"`).
**Лечение**: Создавать отдельные файлы `actions.tsx` для клиентских обработчиков.

## Инцидент №3: SSR Import Errors (jsPDF)
**Проблема**: `ReferenceError: window is not defined` при импорте библиотек, работающих с DOM, на сервере.
**Причина**: Прямой импорт в верхней части файла.
**Урок**: Библиотеки вроде `jspdf`, `xlsx` должны импортироваться динамически (`await import`) внутри функций или `useEffect`.
**Лечение**: Использовать `async` функции для экспорта файлов.

## Инцидент №4: Placeholder Logic
**Проблема**: Использование `// ...` в `replace_file_content` привело к удалению рабочего кода.
**Урок**: Никогда не использовать сокращения в инструментах редактирования файлов.
**Лечение**: Только полная замена блоков или использование `multi_replace` с точными границами.

## Инцидент №5: Кириллица в PDF (jsPDF)
**Проблема**: Вместо русского текста в PDF отображаются пустые места или "кракозябры".
**Причина**: Стандартные шрифты jsPDF (Helvetica) не поддерживают кириллицу.
**Урок**: Для 100% поддержки любых языков и сохранения дизайна лучше использовать `html2canvas` для рендеринга HTML в изображение внутри PDF.
**Лечение**: Использовать `html2canvas` + `jsPDF` для экспорта "пиксель-в-пиксель".

## Инцидент №6: Ошибка сортировки (NaN Date)
**Проблема**: Товары пропадают или сортируются некорректно.
**Причина**: Попытка получить `getTime()` от `undefined` или `null` даты в `sort()`.
**Урок**: Всегда использовать fallback значение (например, `0`) при создании `new Date()` в функциях сортировки.
**Лечение**: `new Date(item.date ?? 0).getTime()`.

## Инцидент №7: Ошибка типов шрифта (Google Fonts Subsets)
**Проблема**: Ошибка сборки `Type error: Type '"cyrillic"' is not assignable to type '"latin" | "latin-ext"'`.
**Причина**: Попытка использовать `cyrillic` для шрифта `DM_Sans`, который в данной версии `@next/font` не поддерживает это подмножество.
**Урок**: Перед деплоем всегда запускать `npm run build` локально для проверки совместимости типов Google Fonts.
**Лечение**: Удалено `cyrillic` из конфигурации `DM_Sans` в `layout.tsx`.

## Инцидент №8: Ошибка типов ZodError
**Проблема**: `Property 'errors' does not exist on type 'ZodError<unknown>'`.
**Причина**: В используемой версии `zod` список ошибок находится в поле `issues`, а не `errors`.
**Урок**: При обработке ZodError всегда использовать `issues` для получения деталей валидации.
**Лечение**: Заменено `error.errors` на `error.issues` в `src/app/proposals/actions.ts`.
